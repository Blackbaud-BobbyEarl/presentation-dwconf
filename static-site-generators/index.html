What is a static site generator?

Tool to transform content into HTML website as build time, as opposed to run time, while still supporting the following concepts:

  • Templating
  • Multiple input types: HTML, Markdown, JSON, YAML
  • Multiple output types: HTML, RSS, JSON, etc.

What are they good at / not good at?

The Good

  • Performance at build time not run time
  • Version control
  • Security (compared to CMS, not webserver)
  • Deployable to any server / environment

The Bad

  • No real time / targeted content
  • No central mechanism to add new content
  • No built in commenting system

Mostly solvable BTW

How to choose a static site generator.

Visit staticsitegenerators.net and roll the dice.

Jekyll

  • Built with Ruby
  • Uses Liquid templating
  • Well documented
  • Under active development
  • Somewhat tough to install on Windows

Assemble

  • Built with NodeJS
  • Uses Handlebars templating
  • Well documented, but not all nuances
  • Easy to install on any system

GitHub Pages Highlights

  • For user/organization, create (user/organization).github.io repo
  • For repo, create gh-pages branch
  • Can be automated via repo's settings
  • Doesn't have to use Jekyll to use GitHub Pages
